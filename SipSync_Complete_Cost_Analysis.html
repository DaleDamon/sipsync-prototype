1. Current System Architecture & Usage Patterns

Database Operations Per User Journey

Understanding Firestore costs requires analyzing typical user behavior:

Scenario 1: New User Signs Up & Takes Quiz

  1. Create user account: 1 write
  2. Submit quiz: 1 write (save preferences)
  3. View quiz results: 1 read (user data)
  4. Navigate to pairing discovery: 0 operations (uses cached data)
  5. Find wine pairings:
    • 1 read (restaurant data)
    • 50 reads (wine subcollection - avg 50 wines per restaurant)
    • 20 reads (food items subcollection)
    • 50 reads (pairings subcollection)
  6. Save favorite pairing: 1 write

Total per new user: ~123 reads, 3 writes

Scenario 2: Returning User Browses Wines

  1. Load profile page: 1 read (user), 50 reads (pairing history), 5 reads (visited restaurants)
  2. Browse wines at restaurant: 1 read (restaurant), 50 reads (wines), 20 reads (foods), 50 reads (pairings)
  3. Save new pairing: 1 write

Total per returning session: ~177 reads, 1 write

Scenario 3: Restaurant Uploads Wine List (CSV)

  1. Upload 50-wine CSV: 0 reads, 50 writes (batch import)
  2. Daily update (10% churn): 0 reads, 5 writes

Total per restaurant onboarding: 50 writes

Total per daily update: 5 writes (avg)

‚ö†Ô∏è Critical Optimization Opportunity: Caching

The largest cost driver is network egress from repeated wine list queries. Implementing a CDN or Redis cache layer can reduce this by 70-80%:

  • Without caching: $134/month egress at 5,000 restaurants
  • With CloudFlare CDN (free tier): ~$30/month egress (75% reduction)
  • With Redis cache: $15/month Redis + $30/month egress = $45/month total

Recommended: Add caching layer at Growth stage to cap Firestore costs.

10. Cost Optimization Recommendations

High-Impact Optimizations (Implement Immediately)

  1. Implement Wine Caching (Redis)
    • Cost: $10-15/month
    • Savings: $100/month at scale (Firestore egress)
    • ROI: 7-10x return
    • When: Implement at 100-200 restaurants
  2. Switch to Email/Password Auth
    • Cost: $0 (free)
    • Savings: $150/month at scale (eliminate SMS)
    • Trade-off: Slightly higher friction than phone auth
    • When: Implement in MVP or shortly after launch
  3. Use AI for Wine List Uploads
    • Cost: $8/month at scale
    • Savings: $22,500/month in onboarding labor
    • ROI: 2,800x return
    • When: Critical for MVP - don't launch without it

Medium-Impact Optimizations (Implement at Growth Stage)

  1. Optimize Firestore Queries
    • Use composite indexes for common queries
    • Implement pagination (limit results to 20-50 wines per query)
    • Cache user preferences in localStorage to reduce reads
    • Estimated savings: 20-30% reduction in reads
  2. CDN for Static Assets
    • Use CloudFlare Pages (free) for frontend hosting
    • Reduces bandwidth costs and improves load times
    • Savings: $20-40/month vs paid hosting
  3. Batch Wine Updates
    • Allow restaurants to schedule batch updates (weekly vs daily)
    • Reduces write operations and AI processing
    • Savings: 30-50% reduction in daily update costs

Low-Priority Optimizations (Consider at Scale)

  1. Self-Service Knowledge Base
    • Build FAQ and video tutorials for restaurant onboarding
    • Reduces support tickets by 20-40%
    • Savings: $3,000-6,000/month in CS labor at scale
  2. AI-Powered Support Bot
    • Chatbot for common restaurant questions
    • Cost: $50-100/month (service fee)
    • Savings: $2,000-4,000/month in CS labor
